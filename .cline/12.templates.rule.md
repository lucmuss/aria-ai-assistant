# Django Templates: Canonical Location

## Rule
- All Django HTML/TXT templates MUST live under `src/templates/`.
- Do not create per-app template folders like `src/<app>/templates/`.

## Rationale
- Single, predictable template root simplifies refactors, grepping, and deployment packaging.
- Avoids duplication/confusion between project-level templates and app-level templates.

## Requirements
- `src/redflag_project/settings.py` must keep `TEMPLATES[0]["DIRS"] = [BASE_DIR / "templates"]` (where `BASE_DIR` is `src/`).
- When adding templates, place them in a path that matches the Django lookup convention under `src/templates/`:
  - Example: `src/templates/account/login.html`
  - Example: `src/templates/emails/welcome_email.txt`

## Allowed Exceptions
- None. If a 3rd-party app requires overriding templates, override them in `src/templates/<app_name>/...` instead of creating `src/<app>/templates/`.
